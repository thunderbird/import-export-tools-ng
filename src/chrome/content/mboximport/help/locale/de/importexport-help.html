ImportExportTools NG – Übersicht und Menüs

ImportExportTools NG ist eine Weiterentwicklung der ursprünglichen Arbeit von Paolo „Kaosmos“. Die Erweiterung bietet vielfältige Import- und Exportfunktionen und unterstützt verschiedene Dateiformate. Hinweis: Einige Formate werden nicht von allen Funktionen unterstützt.
- mbox
- EML
- Mailverzeichnis
- HTML
- Klartext
- CSV (Komma-getrennte Werte)
- PDF

Der Großteil der Funktionalität dreht sich um Nachrichten, es werden aber auch der Import und Export von Thunderbird-Profilen unterstützt. Exporte können zusätzlich eine HTML- oder CSV-Indexdatei enthalten. Auch die Sicherung des Thunderbird-Profils ist möglich.

Der Zugriff auf die IETng-Funktionen erfolgt über verschiedene, von der Erweiterung installierte Menüs. Die Funktionen sind kontextsensitiv. Folgende Menüs wurden hinzugefügt:
- Der Ordnerbereichskontext (Ordneroperationen)
- Der Nachrichtenlistenkontext (ausgewählte Nachrichten)
- Das Menü „Tools“ (Profil- und Sicherungsvorgänge)

Einige Optionen, die über das Menü „Extras“ zugänglich sind, befinden sich noch nicht im Hamburger-Menü, werden aber in einer zukünftigen Version verfügbar sein. Diese Funktionen sind auch über die neue Symbolleistenschaltfläche verfügbar. Jedes Menü und jede Schaltfläche verfügt außerdem über einen Hilfe- und einen Optionsmenüpunkt, über den Sie den Optionsdialog öffnen können.

Unterstützung:

- ImportExportTools NG-Version

Benutzerdefiniertes Datumsformat

Die Option „Benutzerdefiniertes Datum“ ermöglicht die vollständige Kontrolle über das Datumsformat beider Dateien im Index. Sowohl das Standardformat „Datum“ als auch das neue benutzerdefinierte Format „Datum (benutzerdefiniert)“ stehen in den Dateinamen-Dropdown-Menüs zur Verfügung. Das benutzerdefinierte Datum kann auch mit dem Token ${date_custom} verwendet werden, das in erweiterten benutzerdefinierten Dateinamen zum Einsatz kommt. Das ursprüngliche (Standard-)Datumsformat ist:
YYYYMMDDHHmm : 202011060808

Das benutzerdefinierte Datumsformat verwendet Prozentzeichenfolgen, die der Unix-Funktionalität von strftime() entsprechen. Es basiert auf einer JavaScript-Portierung von strftime() , einer Funktion zur Formatierung von Datum und Uhrzeit.
Portierung von strftime() durch TH Doan (https://thdoan.github.io/strftime/)

  • %c - formatierte Zeichenkette ist etwas anders
  • %D - nicht implementiert (verwenden Sie %m/%d/%y oder %d/%m/%y )

  • %E - nicht implementiert
  • %h - nicht implementiert (verwenden Sie %b )
  • %k – Leerzeichen wird nicht hinzugefügt
  • %n - wie %m , aber ohne führende Null (verwenden Sie \n für einen Zeilenumbruch)
  • %O - nicht implementiert
  • %r - nicht implementiert (verwenden Sie %I:%M:%S %p )
    • ${subject} Betreff der Nachricht
  • %R - nicht implementiert (verwenden Sie %H:%M )
  • %t - nicht implementiert (verwenden Sie \t )
  • %T - nicht implementiert (verwenden Sie %H:%M:%S )
  • %U - nicht implementiert
  • %W - nicht implementiert
  • %+ - nicht implementiert
  • %% - nicht implementiert (verwenden Sie % )

Neue und experimentelle Einstellungen (Optionen)

Es gibt mehrere neue Funktionen und Optionen, die im Optionsdialog noch nicht konfiguriert werden können. Sobald diese Funktionen stabil sind, werden sie der Benutzeroberfläche hinzugefügt.
Um diese Funktionen zu nutzen, müssen sie im Konfigurationseditor eingestellt werden. Sie erreichen den Editor über das Menü „Extras/Optionen“, wählen dann den Bereich „Erweitert“ und finden unten die Schaltfläche „Konfigurationseditor“. Wenn Sie „importexporttoolsng“ in die Suchleiste eingeben, werden Ihnen alle Erweiterungseinstellungen angezeigt. Anschließend können Sie die verschiedenen Einstellungen nach Bedarf bearbeiten.

importexporttoolsng.export.index_date_custom_format

Diese Einstellung verwendet dieselbe vollständige Formatierung wie die Indexdateien. Standardmäßig ist sie eine leere Zeichenkette. Wenn sie eine formatierte Datums-/Zeitzeichenkette enthält, ändert sich dadurch das Datumsformat des Index.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Beide Einstellungen verwenden dasselbe erweiterte Dateibenennungsformat wie die exportierten Nachrichten. Die erste Einstellung gilt für externe Anhänge, die zweite für eingebettete (Inline-)Anhänge.

importexporttoolsng.experimental.use_delivery_date

Normalerweise wird in Thunderbird für eine Nachricht das Datum verwendet, das vom E-Mail-Programm des Absenders festgelegt wurde. Wenn Sie diese Option auf „true“ setzen, wird das Empfangsdatum im E-Mail-Programm des Empfängers verwendet. Dieses Datum wird sowohl für Dateinamen als auch für das Datum in den Indexdateien verwendet.

Grundlegende, globale Hotkey-Funktion (Tastenkombinationen)

importexporttoolsng.experimental.hot_keys

Diese neue Einstellung ermöglicht es dem Benutzer, bis zu 10 Tastenkombinationen festzulegen. Die Tastenkombinationen können die meisten Funktionen von ImportExportTools NG sowie weitere Thunderbird-Funktionen ausführen. Die Einstellungszeichenfolge ist als JSON-Objekt formatiert. Format und Schlüssel sind wie folgt:

id - der Schlüsselindex (1-10)
key - der Schlüssel (einzelnes Zeichen)
keycode - der 'VK_*'-definierte Tastencode
Entweder der Schlüssel oder der Schlüsselcode kann angegeben werden, aber nicht beides.
modifiers - die Tastenmodifikatoren, durch Leerzeichen getrennte Liste ('Strg', 'Umschalt', 'Alt', 'Beschleunig')

'all' – alle Fenster unter Messenger, Verfassen, Suchen
'messenger' – das Hauptfenster des Messengers und seine Nachfolger
'compose' – das Nachrichtenfenster
