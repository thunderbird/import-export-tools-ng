Notes de version : Nouvelle version majeure v15

Notes, fonctionnalités et modifications de la version 15. Veuillez lire attentivement.

La version 15 réécrit entièrement les fonctions d'exportation des messages, qu'il s'agisse des dossiers ou des messages sélectionnés. C'est la première étape d'une transition technologique moderne, principalement basée sur les extensions Web. La nouvelle architecture offre une nette amélioration, permettant de nouvelles fonctionnalités et des performances accrues.

Pour cette première version v15, les formats suivants ont été implémentés :

- EML
- HTML et pièces jointes
- PDF et pièces jointes
- Texte brut et pièces jointes

Outre la nouvelle fonctionnalité d'exportation de PDF avec pièces jointes, une nouvelle fenêtre d'état d'exportation et une nouvelle option de notification du système d'exploitation sont également disponibles. Voici la liste des fonctionnalités :

Exportation PDF avec pièces jointes

Il s'agit d'une option très demandée. Une limitation importante subsiste par rapport à l'exportation HTML : le tableau des pièces jointes ne contient pas de liens cliquables. Jusqu'à présent, il n'a pas été possible de générer un document avec des liens relatifs. Le convertisseur PDF et/ou les visionneuses convertissent tous les liens file:// en liens absolus, ce qui les rend inutilisables. J'espère qu'une solution sera trouvée à l'avenir.

Indice

Le nouveau fichier d'index vous permet de trier les colonnes en cliquant sur n'importe quel en-tête.

Structures d'exportation :

- Toutes les pièces jointes dans un seul répertoire
Si vous utilisez un nom statique comme « Pièces jointes », toutes les pièces jointes seront placées dans ce répertoire. Auparavant, cela aurait créé des dossiers tels que Pièces jointes, Pièces jointes-1, Pièces jointes-2, etc.

- Pièces jointes enregistrées dans le même répertoire que les messages.
Dans cette nouvelle option de l'onglet Noms de fichiers, aucun répertoire de pièces jointes n'est créé car elles se trouvent toutes dans le répertoire des messages.

- Les répertoires de pièces jointes ont désormais la date du message définie.
Pour le tri des pièces jointes, la date de modification sera désormais définie sur la date d'envoi du message correspondant.

- Structure expérimentale « Withatts »
Cette structure regroupe chaque message et toutes ses pièces jointes dans un répertoire de pièces jointes formaté. En règle générale, vous pouvez définir le format des noms de fichiers selon vos préférences, par exemple ${subject}, et celui des répertoires également selon ${subject}. Cela créera un répertoire pour chaque message, contenant à la fois le message et ses pièces jointes. Il est possible de trier ces répertoires par date ou par nom.

Pour tester cette structure expérimentale, insérez le mot :
avecatts
dans les options de journalisation de l'onglet Divers. (Proposition de @Rayerd)

Options de journalisation :

Les nouvelles options de journalisation se trouvent dans l'onglet Divers. Elles sont définies sous forme de liste de types de journalisation séparés par des espaces. Elles génèrent divers types de messages de journalisation et de débogage dans la console de débogage, accessible via Ctrl+Maj+J. Les types de journalisation peuvent être combinés pour produire des résultats cumulatifs. Voici la liste des types de journalisation :
- msgs : État de sauvegarde des messages de base
- msgs2 : Sortie de débogage plus détaillée pour les messages
- résumé : Un résumé de l'exportation
- msgparts : Informations de débogage pour les parties de message MIME
- withatts : Une option temporaire pour utiliser la structure combinée des messages et des pièces jointes

Présentation et menus d'ImportExportTools NG

ImportExportTools NG est une extension dérivée du travail original de Paolo « Kaosmos ». Elle offre de multiples fonctions d'importation et d'exportation prenant en charge différents formats de fichiers. Remarque : certains formats ne sont pas compatibles avec toutes les fonctions.
- boîte de réception
- EML
- maildir
- HTML
- texte brut
- CSV (Valeurs séparées par des virgules)
- PDF

Bien que la plupart des fonctionnalités soient liées à la messagerie, l'importation et l'exportation des profils Thunderbird sont également prises en charge. De plus, les exportations peuvent inclure un fichier d'index HTML ou CSV. La sauvegarde du profil Thunderbird est une autre fonctionnalité disponible.

L'accès aux fonctionnalités d'IETng se fait via plusieurs menus installés par l'extension. Ces fonctions sont contextuelles. Des menus sont ajoutés pour :
- Contexte du volet Dossier (opération sur les dossiers)
- Contexte de la liste des messages (messages sélectionnés)
- Le menu Outils (opérations de profil et de sauvegarde)

Les options accessibles via le menu « Outils » ne figurent pas encore dans le menu « Hamburger », mais le seront dans une version ultérieure. Ces fonctions sont également disponibles via le nouveau bouton de la barre d'outils. Chaque menu et bouton dispose d'une option « Aide et options » permettant d'accéder à la boîte de dialogue des options.

Soutien:

Veuillez signaler les problèmes ou les demandes de fonctionnalités dans le dépôt GitHub des extensions : ImportExportTools NG Issues

Pour signaler un problème de bogue, veuillez inclure les informations suivantes afin de faciliter sa résolution :
- Version ImportExportTools NG
- Version Thunderbird
- Système d'exploitation et version
- Sortie de débogage (copier depuis la console de débogage - accessible avec Ctrl+Maj+J)
- Une description des étapes ayant mené au problème

Format de date personnalisé

L'option de date personnalisée permet un contrôle total sur le format de date utilisé pour les deux fichiers de l'index. Le format par défaut « Date » et le nouveau format personnalisé « Date (personnalisée) » sont disponibles dans les listes déroulantes des noms de fichiers. La date personnalisée peut également être utilisée avec le jeton ${date_custom} dans les noms de fichiers personnalisés étendus. Le format de date d'origine (par défaut) est :
AAAAMMJJHHmm : 202011060808

Le format de date personnalisé utilise des séquences de pourcentage, similaires à la fonctionnalité Unix de la fonction strftime() . Il est basé sur une adaptation JavaScript de strftime() , une fonction permettant de formater la date et l'heure.
Portage de strftime() par TH Doan (https://thdoan.github.io/strftime/)

  • %U - non implémenté

Préférences nouvelles et expérimentales (options)

Plusieurs nouvelles fonctionnalités et options ne sont pas encore configurables dans la boîte de dialogue des options. Elles seront ajoutées à l'interface utilisateur une fois stabilisées.
Pour les utiliser, vous devez les configurer dans l'éditeur de configuration. Pour y accéder, utilisez le menu Outils/Options, puis sélectionnez la section Avancé. Un bouton « Éditeur de configuration » se trouve en bas. En saisissant « importexporttoolsng » dans la barre de recherche, vous trouverez toutes les préférences de l'extension. Vous pourrez ensuite modifier les différentes préférences ci-dessous selon vos besoins.

importexporttoolsng.export.index_date_custom_format

Cette préférence utilise le même formatage complet que les fichiers d'index. Par défaut, il s'agit d'une chaîne vide ; si elle contient une chaîne de date/heure formatée, le format de date de l'index sera modifié.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Ces deux préférences utilisent le même type de format de nommage étendu que celui utilisé pour les messages exportés. La première concerne les pièces jointes externes, la seconde les pièces jointes intégrées (en ligne).

importexporttoolsng.experimental.use_delivery_date

Normalement, la date d'un message dans Thunderbird est celle définie par le client de messagerie de l'expéditeur. En activant cette option, la date de réception enregistrée par le client de messagerie du destinataire sera utilisée. Cette date servira à la fois pour les noms de fichiers et pour la date dans les fichiers d'index.
