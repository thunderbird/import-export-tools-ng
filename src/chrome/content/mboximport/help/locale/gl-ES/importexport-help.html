ImportExportTools NG Visión xeral e menús

ImportExportTools NG é unha extensión de seguimento baseada no traballo orixinal de Paolo "Kaosmos". A extensión ofrece varias funcións de importación e exportación que admiten varios formatos de ficheiro diferentes, incluíndo: Nota: algúns formatos non son compatibles con todas as funcións.
- mbox
- EML
- maildir
- HTML
- texto plano
- CSV (valores separados por comas)
- PDF

Aínda que a maioría das funcións xiran en torno ás mensaxes, tamén hai soporte para a importación e exportación de perfís Thunderbird. Ademais, as exportacións tamén poden incluír un ficheiro de índice HTML ou CSV. A copia de seguridade do perfil Thunderbird é outra característica.

O acceso á funcionalidade IETng faise a través de varios menús diferentes instalados pola extensión. As funcións son sensibles ao contexto. Engádense menús para:
- O contexto do panel de cartafoles (operación do cartafol)
- O contexto da lista de mensaxes (mensaxes seleccionadas)
- O menú Ferramentas (operacións de cartafol e perfil)

Hai moitas opcións ás que se pode acceder na entrada engadida de "Ferramentas / Opcións de complemento" para as opcións de extensión. Cada un dos menús tamén ten un elemento do menú Opcións para acceder ao diálogo de opcións.
As novas opcións están documentadas aquí, o conxunto completo documentarase posteriormente.

Formato de data personalizado

A opción de data personalizada permite o control completo sobre o formato de data usado para ambos ficheiros do índice. Tanto o formato predeterminado "Data" como o novo formato personalizado "Data (personalizado)" están dispoñibles nos despregables do nome do ficheiro. A data personalizada tamén se pode usar co token $ {date_custom} empregado nos nomes de ficheiro personalizados ampliados. O formato de data orixinal (predeterminado) é:
AAAAMMDDHHmm: 202011060808

O formato de data personalizado emprega% secuencias que coinciden coa funcionalidade como strftime() Unix. Baséase nun porto JavaScript de strftime() , unha función para formatar a data e a hora.
Port of strftime () por TH Doan (https://thdoan.github.io/strftime/)

Descricións de secuencias de conversión

Secuencia Descrición
%a Nome abreviado do día da semana.
%A Nome completo do día da semana.
%b Nome do mes abreviado.
%B Nome do mes completo.

%g Como %G , pero sen século, é dicir, cun ano de dous díxitos (00-99).
%H Hora como número decimal usando un reloxo de 24 horas (rango de 00 a 23). Vexa tamén %k .

Notas de compatibilidade:

  • %c - a cadea formatada é lixeiramente diferente
  • %D - non implementado (use %m/%d/%y ou %d/%m/%y )
  • Non se engade %e - espazo
  • %E - non implementado
  • %h - non implementado (use %b )
  • Non se engade %k - espazo
  • %n - como %m , pero sen cero inicial (use \n para nova liña)
  • %O - non implementado
  • %r - non implementado (use %I:%M:%S %p )
    • ${subject} O asunto da mensaxe
  • %R - non implementado (use %H:%M )
  • %t - non implementado (usar \t )
  • %T - non implementado (use %H:%M:%S )
  • %U - non implementado
  • %W - non implementado
  • %+ - non implementado
  • %% - non implementado (use % )

Preferencias novas e experimentais (opcións)

Hai varias funcións e opcións novas que aínda non se poden configurar no diálogo de opcións. Unha vez estabilizadas estas funcións, engadiranse á interface de usuario.
Para usalos deben estar configurados dentro do Editor de configuración. Para acceder ao editor, use o menú Ferramentas / Opcións e seleccione a sección Avanzado e haberá un botón "Configurar editor" na parte inferior. Se escribes "importexporttoolsng" na barra de busca, obterás todas as preferencias de extensión. A continuación, pode editar as distintas preferencias a continuación segundo sexa necesario.

importexporttoolsng.export.index_date_custom_format

Esta preferencia segue o mesmo formato completo que se usará para os ficheiros de índice. É unha cadea baleira por defecto, se ten unha cadea de data / hora con formato, entón cambiará o formato de data do índice.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Estas dúas preferencias usan o mesmo tipo de formatos de nome de ficheiros estendidos que usa o nome do ficheiro das mensaxes exportadas. O primeiro é para os anexos externos, o segundo é para os anexos incrustados (en liña).

importexporttoolsng.experimental.use_delivery_date

Normalmente a 'Data' para unha mensaxe no Thunderbird é a data establecida polo cliente de correo do remitente. Ao configurar esta opción como "verdadeiro" utilízase a data de recepción no cliente de correo do destinatario. Esta data usarase tanto para os nomes de ficheiros como para a data nos ficheiros de índice.

Función de tecla de acceso global básica (atallos)

importexporttoolsng.experimental.hot_keys

Esta nova preferencia permite ao usuario configurar ata 10 atallos de teclado. Os atallos poden realizar a maioría das funcións ImportExportTools NG, así como outras funcións do Thunderbird. A cadea de preferencias está formatada como un obxecto formatado JSON. O formato e as teclas son os seguintes:

Exemplos de entradas de teclas rápidas (matriz de obxectos JSON)

id - o índice de chave (1-10)
key : a tecla (carácter único)
keycode : o código de clave "VK_ *"
Pode incluírse calquera clave ou código de clave, pero non ambas as dúas
modifiers - os modificadores clave, lista separada por espazo ('control', 'shift', 'alt', 'accel')
oncommand : unha cadea de comando directa
contexts : o contexts da xanela onde a tecla está activa
'all' : todas as fiestras de Messenger, compoñer, buscar
'messenger' : a xanela principal de mensaxería e os seus descendentes
'compose' : a xanela de redacción da mensaxe
'search' : o diálogo de busca
