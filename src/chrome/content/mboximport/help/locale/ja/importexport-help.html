ImportExportTools NGの概要とメニュー

ImportExportTools NG は、Paolo "Kaosmos" 氏のオリジナル作品を基にした拡張機能です。この拡張機能は、複数のインポートおよびエクスポート機能を提供し、以下を含むいくつかの異なるファイル形式をサポートしています。注: 一部の形式は、すべての機能でサポートされているわけではありません。
- mbox
- EML
- メールディレクトリ
- HTML
- プレーンテキスト
- CSV（カンマ区切り値）
- PDF

機能の大部分はメッセージに関するものですが、Thunderbirdプロファイルのインポートとエクスポートにも対応しています。さらに、エクスポート時にはHTMLまたはCSV形式のインデックスファイルを含めることも可能です。Thunderbirdプロファイルのバックアップ機能も備えています。

IETngの機能にアクセスするには、拡張機能によってインストールされる複数の異なるメニューを使用します。これらの機能は状況に応じて変化します。メニューは以下の目的で追加されます。
- フォルダーペインのコンテキスト（フォルダー操作）
- メッセージリストのコンテキスト（選択されたメッセージ）
- ツールメニュー（プロファイルおよびバックアップ操作）

「ツール」メニューにあるオプションの中には、「ハンバーガー」メニューには含まれていないものがありますが、今後のバージョンで追加される予定です。これらの機能は、新しいツールバーボタンからも利用できます。各メニューとボタンには、オプションダイアログにアクセスするための「ヘルプ」と「オプション」メニュー項目も用意されています。

カスタム日付形式

カスタム日付オプションを使用すると、インデックス内の両方のファイルで使用される日付フォーマットを完全に制御できます。ファイル名のドロップダウンリストには、デフォルトのフォーマット「日付」と新しいカスタムフォーマット「日付（カスタム）」の両方が表示されます。カスタム日付は、拡張カスタムファイル名で使用される${date_custom}トークンでも使用できます。元の（デフォルトの）日付フォーマットは次のとおりです。
YYYYMMDDHHmm : 202011060808

カスタム日付フォーマットは、Unixのstrftime()関数と同様の機能を持つ%シーケンスを利用します。これは、日付と時刻をフォーマットする関数であるstrftime()のJavaScript移植版に基づいています。
TH Doan氏によるstrftime()の移植版（https://thdoan.github.io/strftime/）

順序 説明

%A 曜日の正式名称。
%b 月名の略称。

  • %R - 未実装（ %H:%Mを使用してください）
  • %t - 未実装（ \tを使用してください）
  • %T - 未実装（ %H:%M:%Sを使用してください）
  • %U - 未実装
  • %W - 未実装
  • %+ - 未実装
  • %% - 未実装（ %を使用してください）

${subject}-${date}-${index} - デフォルトの形式
${prefix}_${date_custom}_${sender} - デフォルトの日付形式ではなく、カスタム日付形式を使用します
${subject}-GmailMsg-${date}-${index} - カスタムテキスト付き

新規および試験的な設定（オプション）

オプションダイアログではまだ設定できない新機能やオプションがいくつかあります。これらの機能が安定次第、ユーザーインターフェースに追加されます。
これらを使用するには、設定エディタで設定する必要があります。エディタにアクセスするには、「ツール」→「オプション」メニューから「詳細設定」セクションを選択し、下部にある「設定エディタ」ボタンをクリックします。検索バーに「importexporttoolsng」と入力すると、すべての拡張機能の設定が表示されます。必要に応じて、以下の各種設定を編集できます。

importexporttoolsng.export.index_date_custom_format

この設定は、インデックスファイルに使用される完全な書式設定に従います。デフォルトでは空の文字列ですが、書式設定された日付/時刻文字列を指定すると、インデックスの日付形式が変更されます。

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

これらの設定はどちらも、エクスポートされたメッセージのファイル名と同じ種類の拡張ファイル命名形式を使用します。1つ目は外部添付ファイル用、2つ目は埋め込み（インライン）添付ファイル用です。

importexporttoolsng.experimental.use_delivery_date

通常、Thunderbirdにおけるメッセージの「日付」は、送信者のメールクライアントで設定された日付です。このオプションを「true」に設定すると、受信者のメールクライアントで受信された日付が使用されます。この日付は、ファイル名とインデックスファイルの日付の両方に使用されます。

基本的なグローバルホットキー（ショートカット）機能

importexporttoolsng.experimental.hot_keys

この新しい設定では、最大10個のキーボードショートカットを設定できます。ショートカットは、ImportExportTools NGのほとんどの機能に加え、Thunderbirdのその他の機能も実行できます。設定文字列はJSON形式のオブジェクトとして記述します。形式とキーは以下のとおりです。

ホットキー入力例（JSONオブジェクトの配列）

id - キーインデックス（1～10）
key - キー（1文字）
keycode - 'VK_*' 定義キーコード
キーまたはキーコードのいずれか一方のみを含めることができますが、両方を含めることはできません。
modifiers - キー修飾キーをスペースで区切ったリスト（'control'、'shift'、'alt'、'accel'）
oncommand - 直接コマンド文字列
contexts - キーがアクティブになっているウィンドウコンテキスト
'all' - メッセンジャー、作成、検索の下にあるすべてのウィンドウ
'messenger' - メインのメッセンジャーウィンドウとその子孫
'compose' - メッセージ作成ウィンドウ
'search' - 検索ダイアログ

[{"id": "1", "key": "K", "modifiers": "control shift", "oncommand": "alert('Compose alert')", "contexts": "compose"}, {"id": "2", "keycode: "VK_F1", "modifiers": "control shift", "oncommand": "alert('testD')", "contexts": "all"}]

[{"id": "1", "key": "P", "modifiers": "control shift", "oncommand": "goDoCommand('cmd_printpreview')"}, {"id": "2", "key": "D", "modifiers": "control shift", "oncommand": "exportSelectedMsgs(5)"}]
