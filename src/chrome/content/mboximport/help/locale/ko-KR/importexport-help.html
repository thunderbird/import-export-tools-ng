릴리스 노트: 주요 업데이트 v15 버전 출시

버전 15의 주요 특징, 기능 및 변경 사항입니다. 자세히 읽어주시기 바랍니다.

v15는 폴더 내보내기 및 선택한 메시지 내보내기 기능을 완전히 새롭게 재작성한 버전입니다. 이는 최신 웹 확장 기술을 도입하는 첫 번째 단계입니다. 새로운 디자인은 아키텍처를 대폭 개선하여 새로운 기능과 향상된 성능을 제공합니다.

첫 번째 v15 릴리스에서는 다음과 같은 형식이 구현되었습니다.

- EML
- HTML 및 첨부 파일
- PDF 및 첨부 파일
- 일반 텍스트 및 첨부 파일

첨부 파일이 포함된 PDF 내보내기 기능 외에도 새로운 내보내기 상태 창과 새로운 운영 체제 알림 옵션이 추가되었습니다. 기능 목록은 다음과 같습니다.

첨부파일이 포함된 PDF 내보내기

이 기능은 오랫동안 요청되어 온 기능입니다. HTML 내보내기와 비교했을 때 중요한 제약 사항이 하나 있습니다. 첨부 파일 표에 클릭 가능한 링크가 없다는 것입니다. 현재로서는 상대 링크를 포함한 출력을 생성할 수 없습니다. PDF 변환기나 뷰어가 모든 file:// 링크를 절대 링크로 변환해 버리는데, 이는 유용하지 않습니다. 앞으로 이 문제를 해결할 수 있는 방법이 생기기를 바랍니다.

색인

새로운 인덱스 파일을 사용하면 헤더를 클릭하여 열을 정렬할 수 있습니다.

수출 구조

- 모든 첨부 파일이 하나의 디렉토리에 있습니다
"Attachments"와 같이 고정된 단어를 사용하면 모든 첨부 파일이 해당 디렉터리에 저장됩니다. 이전에는 이렇게 하면 Attachments, Attachments-1, Attachments-2 등과 같이 생성되었습니다.

- 첨부 파일은 메시지와 동일한 디렉토리에 저장됩니다.
파일 이름 탭에서 찾을 수 있는 이 새로운 옵션을 사용하면 모든 첨부 파일이 메시지 디렉터리에 있으므로 첨부 파일 디렉터리가 생성되지 않습니다.

- 첨부파일 디렉터리에 메시지 날짜가 설정됩니다.
이제 첨부 파일 디렉터리를 정렬할 때 수정 날짜가 해당 메시지의 전송 날짜로 설정됩니다.

- 실험적인 "withatts" 구조
이 구조는 각 메시지와 모든 첨부 파일을 지정된 형식의 첨부 파일 디렉터리에 결합합니다. 일반적인 설정은 파일 이름 형식을 원하는 형식(예: ${제목})으로, 디렉터리 형식도 ${제목}으로 설정하는 것입니다. 이렇게 하면 각 메시지에 대한 디렉터리가 생성되며, 이 디렉터리들은 메시지와 첨부 파일을 모두 포함합니다. 디렉터리는 날짜 또는 이름순으로 정렬할 수 있습니다.

이 실험적인 구조를 사용해 보려면 다음 단어를 넣어 보세요.
withatts
기타 탭의 로깅 옵션에 추가합니다. (@Rayerd 님의 제안)

로깅 옵션

새로운 로깅 옵션은 '기타' 탭에 있습니다. 이 옵션은 공백으로 구분된 로그 유형 단어 목록으로 설정됩니다. 이러한 옵션은 Ctrl+Shift+J를 눌러 접근할 수 있는 디버그 콘솔에 다양한 로그 및 디버그 출력을 생성합니다. 로그 유형은 조합하여 누적 출력을 생성할 수 있습니다. 로그 유형은 다음과 같습니다.
- msgs : 기본 메시지 저장 상태
- msgs2 : 메시지에 대한 보다 자세한 디버그 출력
- summary : 수출 요약
- msgparts : MIME 메시지 부분에 대한 디버그 정보
- withatts : 메시지와 첨부파일을 결합한 구조를 사용하는 임시 옵션입니다.

메뉴

새로운 기능은 현재 기존 기능 위에 추가되며 (v15) 접두사가 붙습니다.

새로운 기능이 안정화되면 기존 코드는 더 이상 사용되지 않을 예정입니다. 이는 주요 릴리스에 대한 신중한 접근 방식입니다. 새로운 v15 기능을 사용해 주시고, 문제가 발생하면 여기 GitHub 저장소에 보고해 주세요.

ImportExportTools NG 개요 및 메뉴

ImportExportTools NG는 Paolo "Kaosmos"의 원작을 기반으로 개발된 확장 프로그램입니다. 이 확장 프로그램은 다양한 파일 형식을 지원하는 여러 가져오기 및 내보내기 기능을 제공합니다. 참고: 일부 형식은 모든 기능에서 지원되지 않을 수 있습니다.
- mbox
- EML
- 메일디렉터
- HTML
- 평문
CSV(쉼표로 구분된 값)
- PDF

대부분의 기능은 메시지와 관련되어 있지만, 썬더버드 프로필 가져오기 및 내보내기도 지원합니다. 또한 내보내기 시 HTML 또는 CSV 형식의 색인 파일을 포함할 수 있습니다. 썬더버드 프로필 백업 기능도 제공됩니다.

IETng 기능은 확장 프로그램에서 설치한 여러 메뉴를 통해 접근할 수 있습니다. 기능은 상황에 따라 달라집니다. 다음과 같은 기능을 위한 메뉴가 추가되었습니다.
- 폴더 창 컨텍스트(폴더 작업)
- 메시지 목록 컨텍스트(선택된 메시지)
- 도구 메뉴(프로필 및 백업 작업)

"도구 메뉴"에서 접근 가능한 옵션 중 일부는 "햄버거 메뉴"에는 없지만, 향후 버전에서 추가될 예정입니다. 이러한 기능은 새로운 툴바 버튼에서도 사용할 수 있습니다. 각 메뉴와 버튼에는 도움말 및 옵션 메뉴 항목이 있어 옵션 대화 상자에 접근할 수 있습니다.

지원하다:

문제 또는 기능 요청 사항은 확장 프로그램의 GitHub 저장소( ImportExportTools NG Issues) 에 게시해 주세요.

버그 문제를 신고할 때는 문제 해결에 도움이 될 수 있도록 다음 정보를 포함해 주세요.
- ImportExportTools NG 버전
- 썬더버드 버전
- 운영 체제 및 버전
- 디버그 출력 (디버그 콘솔에서 복사 - Ctrl+Shift+J로 접근)
- 문제 발생에 이르게 된 단계에 대한 설명

사용자 지정 날짜 형식

사용자 지정 날짜 옵션을 사용하면 인덱스의 두 파일 모두에 사용되는 날짜 형식을 완벽하게 제어할 수 있습니다. 기본 형식인 "날짜"와 새로운 사용자 지정 형식인 "날짜(사용자 지정)" 모두 파일 이름 드롭다운 메뉴에서 사용할 수 있습니다. 사용자 지정 날짜는 확장된 사용자 지정 파일 이름에 사용되는 ${date_custom} 토큰과 함께 사용할 수도 있습니다. 원래(기본) 날짜 형식은 다음과 같습니다.
YYYYMMDDHHmm : 202011060808

사용자 지정 날짜 형식은 Unix의 strftime() 함수와 유사한 기능을 하는 % 시퀀스를 사용합니다. 이는 날짜와 시간을 형식화하는 함수인 strftime() 의 JavaScript 포트를 기반으로 합니다.
TH Doan이 작성한 strftime() 함수(https://thdoan.github.io/strftime/)를 포팅했습니다.

변환 순서 설명

순서 설명
%a 요일의 약칭.
%A 요일의 정식 명칭.
%b 월 이름의 약자.
%B 월 이름 전체를 쓰세요.
%c 현재 로케일에 대한 기본 날짜 및 시간(UTC) 표시 방식입니다.
%C 세기를 나타내는 숫자(년/100)를 두 자리 정수로 표현합니다.
%d 월의 일(소수점 표기, 범위: 1~31).
%e 해당 월의 일수를 소수점 이하 자릿수(1~31)로 입력하세요.
%F ISO 8601 날짜 형식( %Y-%m-%d 와 동일).
%G ISO 8601은 세기를 십진수로 나타내는 주 단위 연도 표기법입니다. 4자리 연도는 ISO 주 번호에 해당합니다( %V 참조). 이는 %Y 와 동일한 형식과 값을 가지지만, ISO 주 번호가 이전 연도 또는 다음 연도에 해당하는 경우 해당 연도가 사용됩니다.
%g %G 와 비슷하지만 세기가 없고, 즉 두 자리 연도(00-99)를 사용합니다.
%H 24시간제를 사용하여 시간을 소수로 나타냅니다(범위: 00~23). %k 도 참조하십시오.
%I 12시간제를 사용하여 시간을 소수로 나타냅니다(범위: 01~12). %l 도 참조하십시오.
%j 연중 일수를 십진수(001~366)로 나타낸 숫자입니다.
%k 24시간제를 사용하여 시간을 소수로 나타낸 값(범위 0~23). %H 도 참조하십시오.
%l 시간은 12시간제를 사용하는 소수점 숫자(범위 1~12)로 표시됩니다. %I 도 참조하십시오.
%m 월을 소수점 숫자(01~12)로 입력합니다.
%n 월을 소수점 이하 자릿수(1~12)로 입력하십시오.
%M 분을 소수점 이하 자릿수(00~59)로 나타냅니다.
%p 주어진 시간 값에 따라 "AM" 또는 "PM"으로 표시됩니다. 정오는 "PM"으로, 자정은 "AM"으로 처리됩니다.
%P %p 와 같지만 소문자입니다("am" 또는 "pm").
%s 1970년 1월 1일 00시 00분 00초(UTC) 이후 경과된 초 수.
%S 두 번째 값은 소수점 이하 자릿수(00~59)로 표시됩니다.
%u 요일을 소수점(1~7)으로 나타내며, 월요일이 1입니다. %w 도 참조하십시오.
%V ISO 8601에 따른 당해 연도의 주 번호는 10진수(범위 01~53)로 표시되며, 1주는 새해에 최소 4일 이상인 첫 번째 주(즉, 첫 번째 목요일)입니다.
%w 요일을 소수점(0~6)으로 나타내며, 일요일은 0입니다. %u 도 참조하십시오.
%x 현재 지역에서 시간을 제외한 선호하는 날짜 표시 방식입니다.
%X 현재 위치에서 날짜를 제외한 선호하는 시간 표시 방식입니다.
%y 연도를 세기를 제외한 소수점 이하 숫자로 표시합니다(범위: 00~99).
%Y 세기를 포함한 소수점 이하 숫자로 연도를 표기합니다.
%z +hhmm 또는 -hhmm 숫자 시간대(즉, UTC와의 시분 차이).
%Z 시간대 이름 또는 약어.

호환성 관련 참고 사항:

  • %c - 형식화된 문자열이 약간 다릅니다.
  • %D - 구현되지 않음 ( %m/%d/%y 또는 %d/%m/%y 를 사용하세요)
  • %e - 공백이 추가되지 않습니다.
  • %E - 구현되지 않음
  • %h - 구현되지 않음 ( %b 사용)
  • %k - 공백이 추가되지 않습니다.
  • %n - %m 과 비슷하지만 앞에 0이 붙지 않습니다 (줄 바꿈에는 \n 사용하세요).
  • %O - 구현되지 않음
  • %r - 구현되지 않음 ( %I:%M:%S %p 사용)
    • ${subject} 메시지의 제목
  • %R - 구현되지 않음 ( %H:%M 사용)
  • %t - 구현되지 않음 ( \t 사용)
  • %T - 구현되지 않음 ( %H:%M:%S 사용)
  • %U - 구현되지 않음
  • %W - 구현되지 않음
  • %+ - 구현되지 않음
  • %% - 구현되지 않음 ( % 사용)


확장 파일 이름 형식:


확장 파일 이름 형식은 토큰을 사용합니다. 정확한 파일 이름은 토큰과 일반 텍스트 또는 유효한 파일 이름 문자를 조합하여 구성할 수 있습니다. 유효하지 않은 문자는 다음과 같습니다 | * : < > ? \ / " 영어(en-US) 토큰은 모든 로케일에서 사용할 수 있습니다. 로케일별 토큰도 제공되며, 영어 표 아래에 나열되어 있습니다.

예시:

${subject}-${date}-${index} - 기본 형식
${prefix}_${date_custom}_${sender} - 기본 날짜 형식 대신 사용자 지정 날짜 형식을 사용합니다.
${subject}-GmailMsg-${date}-${index} - 일부 사용자 지정 텍스트 포함

새로운 설정 및 실험적 설정(옵션)

옵션 대화 상자에서 아직 설정할 수 없는 몇 가지 새로운 기능과 옵션이 있습니다. 이러한 기능이 안정화되면 사용자 인터페이스에 추가될 예정입니다.
이 기능을 사용하려면 구성 편집기에서 설정을 변경해야 합니다. 편집기에 접근하려면 도구/옵션 메뉴를 클릭한 다음 고급 섹션을 선택하고 하단에 있는 '구성 편집기' 버튼을 클릭하세요. 검색창에 'importexporttoolsng'를 입력하면 모든 확장 프로그램 설정이 표시됩니다. 필요에 따라 아래의 다양한 설정을 편집할 수 있습니다.

importexporttoolsng.export.index_date_custom_format

이 설정은 인덱스 파일에 사용되는 것과 동일한 전체 형식을 따릅니다. 기본값은 빈 문자열이며, 형식화된 날짜/시간 문자열을 입력하면 인덱스 날짜 형식이 변경됩니다.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

이 두 가지 설정 모두 내보낸 메시지의 파일 이름에 사용되는 것과 동일한 유형의 확장 파일 이름 지정 형식을 사용합니다. 첫 번째는 외부 첨부 파일용이고, 두 번째는 내장(인라인) 첨부 파일용입니다.

importexporttoolsng.experimental.use_delivery_date

일반적으로 썬더버드에서 메시지의 '날짜'는 발신자의 메일 클라이언트에서 설정한 날짜입니다. 이 옵션을 'true'로 설정하면 수신자의 메일 클라이언트에서 수신된 날짜가 사용됩니다. 이 날짜는 파일 이름과 인덱스 파일의 날짜 모두에 사용됩니다.

기본, 전역 단축키(바로가기) 기능

importexporttoolsng.experimental.hot_keys

이 새로운 환경설정을 통해 사용자는 최대 10개의 키보드 단축키를 설정할 수 있습니다. 단축키는 ImportExportTools NG의 대부분 기능과 Thunderbird의 다른 기능들을 수행할 수 있습니다. 환경설정 문자열은 JSON 형식의 객체입니다. 형식과 단축키는 다음과 같습니다.

단축키 입력 예시 (JSON 객체 배열)

id - 키 인덱스(1-10)
key - 키(단일 문자)
keycode - 'VK_*'로 정의된 키코드
키 또는 키코드 중 하나만 포함할 수 있으며, 둘 다 포함할 수는 없습니다.
modifiers - 키 수정자 목록(공백으로 구분)('control', 'shift', 'alt', 'accel')
oncommand - 직접 명령 문자열
contexts - 키가 활성화된 창 컨텍스트
'all' - 메신저, 작성, 검색 아래의 모든 창
'messenger' - 메인 메신저 창 및 그 하위 창들
'compose' - 메시지 작성 창
'search' - 검색 대화 상자
