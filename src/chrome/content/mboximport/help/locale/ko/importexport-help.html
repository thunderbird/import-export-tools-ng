ImportExportTools NG 개요 및 메뉴

ImportExportTools NG는 Paolo "Kaosmos"의 원작을 기반으로 한 후속 확장 프로그램입니다. 이 확장 프로그램은 다음을 포함한 여러 가지 파일 형식을 지원하는 여러 가지 가져오기 및 내보내기 기능을 제공합니다. 참고: 일부 형식은 모든 기능에서 지원되지 않습니다.
- 엠박스
- 엠엘엘
- 메일 디렉토리
- HTML
- 평문
- CSV(쉼표로 구분된 값)
- PDF

대부분의 기능이 메시지를 중심으로 돌아가지만 Thunderbird 프로필의 가져오기 및 내보내기도 지원합니다. 또한 내보내기에는 HTML 또는 CSV 인덱스 파일도 포함될 수 있습니다. Thunderbird 프로필의 백업은 또 다른 기능입니다.

IETng 기능에 액세스하는 것은 확장 프로그램에서 설치한 여러 가지 다른 메뉴를 통해 이루어집니다. 기능은 상황에 따라 달라집니다. 다음에 대한 메뉴가 추가되었습니다.
- 폴더 창 컨텍스트(폴더 작업)
- 메시지 목록 컨텍스트(선택된 메시지)
- 도구 메뉴(프로필 및 백업 작업)

"도구 메뉴"에서 액세스할 수 있는 옵션은 "햄버거" 메뉴에 없으며, 향후 버전에 포함될 예정입니다. 이러한 기능은 새 도구 모음 버튼에서도 사용할 수 있습니다. 각 메뉴와 버튼에는 옵션 대화 상자에 액세스하기 위한 도움말 및 옵션 메뉴 항목도 있습니다.

지원하다:

문제나 기능 요청은 확장 GitHub 저장소인 ImportExportTools NG Issues 에 게시해 주세요.

버그 문제에서는 다음 정보를 포함하여 문제를 해결하는 데 도움을 주세요.
- ImportExportTools NG 버전
- 썬더버드 버전
- 운영체제 및 버전
- 디버그 출력(디버그 콘솔에서 복사 - Control-Shift-J로 액세스)
- 문제로 이어지는 단계에 대한 설명

사용자 정의 날짜 형식

사용자 지정 날짜 옵션을 사용하면 인덱스의 두 파일에 사용된 날짜 형식을 완벽하게 제어할 수 있습니다. 기본 형식 "날짜"와 새 사용자 지정 형식 "날짜(사용자 지정)"는 모두 파일 이름 드롭다운에서 사용할 수 있습니다. 사용자 지정 날짜는 확장된 사용자 지정 파일 이름에 사용된 ${date_custom} 토큰과 함께 사용할 수도 있습니다. 원래(기본) 날짜 형식은 다음과 같습니다.
YYYYMMDDH음 : 202011060808

사용자 지정 날짜 형식은 strftime() Unix 유사 기능과 일치하는 % 시퀀스를 활용합니다. 날짜와 시간을 포맷하는 함수인 strftime() 의 JavaScript 포트를 기반으로 합니다.
TH Doan이 만든 strftime() 포팅(https://thdoan.github.io/strftime/)

변환 순서 설명

순서 설명
%a 요일의 약칭입니다.
%A 요일의 전체 이름입니다.
%b 약어화된 월 이름.
%B 전체 월 이름.
%c 현재 로케일에 대한 기본 날짜 및 시간(UTC) 표현입니다.
%C 세기 숫자(년/100)를 2자리 정수로 표현합니다.
%d 해당 월의 날짜를 10진수로 표현합니다(범위: 01~31).
%e 해당 월의 날짜를 10진수로 표현합니다(범위: 1~31).
%F ISO 8601 날짜 형식( %Y-%m-%d 와 동일).
%G 세기를 10진수로 사용한 ISO 8601 주 기반 연도. 4자리 연도는 ISO 주 번호에 해당합니다( %V 참조). 이것은 %Y 와 동일한 형식과 값을 갖지만 ISO 주 번호가 이전 또는 다음 해에 속하는 경우 해당 연도가 대신 사용됩니다.
%g %G 와 비슷하지만 세기가 없고, 2자리 연도(00-99)를 사용합니다.
%H 24시간제(범위 00~23)를 사용한 10진수 형태의 시간. %k 도 참조하세요.
%I 12시간제(범위 01~12)를 사용한 10진수 형태의 시간. %l 도 참조하세요.
%j 일년 중 날짜를 10진수로 표현합니다(범위: 001~366).
%k 24시간제(범위 0~23)를 사용한 10진수 형태의 시간. %H 도 참조하세요.
%l 12시간제 시계를 사용한 10진수 형태의 시간(범위 1~12). %I 도 참조하세요.
%m 10진수로 표현한 월(범위: 01~12).
%n 10진수로 표현한 월(범위: 1~12).
%M 분을 10진수로 표현(범위: 00~59).
%p 주어진 시간 값에 따라 "AM" 또는 "PM". 정오는 "PM"으로, 자정은 "AM"으로 처리됩니다.
%P %p 와 비슷하지만 소문자입니다("am" 또는 "pm").
%s 1970-01-01 00:00:00 +0000(UTC) 이후의 초 수입니다.
%S 10진수로 표현한 초(범위: 00~59).
%u 소수점 형태로 표현한 요일(범위 1~7), 월요일은 1입니다. %w 도 참조하세요.
%V ISO 8601은 현재 연도의 주 번호를 10진수(범위 01~53)로 표현합니다. 여기서 주 1은 새해에 4일 이상 있는 첫 번째 주(즉, 첫 번째 목요일)입니다.
%w 소수점 형태로 표현한 요일(범위 0~6), 일요일은 0입니다. %u 도 참조하세요.
%x 시간을 제외한 현재 로케일에 대한 기본 날짜 표현입니다.
%X 날짜를 제외한 현재 로케일에 대한 기본 시간 표현입니다.
%y 세기 없는 10진수로 표현한 연도(범위: 00~99).
%Y 세기를 포함한 10진수로 표현한 연도입니다.
%z +hhmm 또는 -hhmm 숫자형 시간대(즉, UTC와의 시간 및 분 오프셋).
%Z 시간대 이름 또는 약어.

호환성 참고 사항:

  • %c - 포맷된 문자열이 약간 다릅니다
  • %D - 구현되지 않음( %m/%d/%y 또는 %d/%m/%y 사용)
  • %e - 공백이 추가되지 않습니다.
  • %E - 구현되지 않음
  • %h - 구현되지 않음( %b 사용)
  • %k - 공백이 추가되지 않습니다.
  • %n - %m 과 유사하지만 앞에 0이 없음(줄바꿈에는 \n 사용)
  • %O - 구현되지 않음
  • %r - 구현되지 않음( %I:%M:%S %p 사용)
    • ${subject} 메시지의 제목
  • %R - 구현되지 않음( %H:%M 사용)
  • %t - 구현되지 않음( \t 사용)
  • %T - 구현되지 않음( %H:%M:%S 사용)
  • %U - 구현되지 않음
  • %W - 구현되지 않음
  • %+ - 구현되지 않음
  • %% - 구현되지 않음( % 사용)


확장된 파일 이름 형식:


확장된 파일 이름 포맷은 토큰을 활용합니다. 정확한 파일 이름은 토큰과 모든 텍스트 또는 유효한 파일 이름 문자를 사용하여 구성할 수 있습니다. 유효하지 않은 문자는 다음과 같습니다. | * : < > ? \ / " 영어(en-US) 토큰은 모든 로캘에서 사용할 수 있습니다. 로캘별 토큰도 사용할 수 있으며 영어 표 아래에 나열되어 있습니다.

예시:

${subject}-${date}-${index} - 기본 형식
${prefix}_${date_custom}_${sender} - 기본값 대신 사용자 지정 날짜 형식을 사용합니다.
${subject}-GmailMsg-${date}-${index} - 일부 사용자 지정 텍스트 포함


토큰 설명
${subject} 메시지의 제목. 길이는 제목 길이 옵션에 의해 제한될 수 있습니다.
${sender} 메시지 발신자의 이름.
${recipient} 메시지 수신자의 이름.
${sender_email} 메시지 발신자의 이메일 주소.
${recipient_email} 메시지 수신자의 이메일 주소.
${smart_name} 메시지가 보낸 편지함에 있는 경우 수신자, 그렇지 않은 경우 발신자 이름
${index} 메시지 인덱스 번호.
${prefix} 접두사 텍스트 옵션 값입니다.
${suffix} 접미사 텍스트 옵션 값입니다.
${date} 표준 날짜 형식입니다. 기본 형식은 YYYYMMDDHHmm입니다.
${date_custom} 사용자 지정 날짜 형식. 사용자 지정 날짜 형식 섹션을 참조하세요.

새로운 실험적 선호도(옵션)

옵션 대화 상자 내에서 아직 구성할 수 없는 여러 가지 새로운 기능과 옵션이 있습니다. 이러한 기능이 안정화되면 UI에 추가됩니다.
이를 사용하려면 Config Editor에서 설정해야 합니다. 편집기에 액세스하려면 Tools/Options 메뉴를 사용한 다음 Advanced 섹션을 선택하면 하단에 'Config Editor' 버튼이 있습니다. 검색 창에 'importexporttoolsng'를 입력하면 모든 확장 환경 설정을 얻을 수 있습니다. 그런 다음 필요에 따라 아래의 다양한 환경 설정을 편집할 수 있습니다.

importexporttoolsng.export.index_date_custom_format

이 기본 설정은 인덱스 파일에 사용되는 동일한 전체 서식을 따릅니다. 기본적으로 빈 문자열이며, 서식이 지정된 날짜/시간 문자열이 있는 경우 인덱스 날짜 서식이 변경됩니다.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

이 두 가지 기본 설정은 모두 내보낸 메시지 파일 이름이 사용하는 것과 동일한 유형의 확장 파일 명명 형식을 사용합니다. 첫 번째는 외부 첨부 파일용이고 두 번째는 내장된(인라인) 첨부 파일용입니다.

importexporttoolsng.experimental.use_delivery_date

일반적으로 Thunderbird의 메시지에 대한 '날짜'는 발신자의 메일 클라이언트에서 설정한 날짜입니다. 이 옵션을 '참'으로 설정하면 수신자의 메일 클라이언트에서 수신한 날짜가 활용됩니다. 이 날짜는 인덱스 파일의 날짜와 파일 이름 모두에 사용됩니다.

기본, 글로벌 핫키(단축키) 기능

importexporttoolsng.experimental.hot_keys

이 새로운 환경 설정은 사용자가 최대 10개의 키보드 단축키를 설정할 수 있도록 합니다. 단축키는 대부분의 ImportExportTools NG 기능과 다른 Thunderbird 기능을 수행할 수 있습니다. 환경 설정 문자열은 JSON 포맷 객체로 포맷됩니다. 형식과 키는 다음과 같습니다.

예제 단축키 항목(JSON 객체 배열)

id - 키 인덱스(1-10)
key - 키(단일 문자)
keycode - 'VK_*' 정의 키코드
키 또는 키코드 중 하나만 포함될 수 있지만 둘 다 포함될 수는 없습니다.
modifiers - 키 수정자, 공백으로 구분된 목록('control', 'shift', 'alt', 'accel')
oncommand - 직접 명령 문자열
contexts - 키가 활성화된 창 컨텍스트
'all' - 메신저, 작성, 검색 아래의 모든 창
'messenger' - 메인 메신저 창과 그 하위 창들
'compose' - 메시지 작성 창
'search' - 검색 대화 상자
