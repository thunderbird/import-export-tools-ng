发布说明：v15 版本重大更新

v15 版本更新说明、功能特性及变更内容。请仔细阅读。

v15 版本对邮件导出功能进行了全面重写，包括文件夹导出和选定邮件导出。这是基于 WebExtension 技术的现代化方案的第一阶段。新设计在架构上进行了大幅改进，从而实现了更多新功能并提升了性能。

首个 v15 版本已实现以下格式：

- EML
- HTML 和附件
- PDF文件及其附件
- 纯文本和附件

除了新增的带附件 PDF 导出功能外，还新增了导出状态窗口和操作系统通知选项。功能列表如下：

导出带附件的PDF文件

这是一个用户期待已久的功能。与 HTML 导出相比，PDF 导出存在一个重要的限制：附件表格中的链接不可点击。目前，PDF 还无法生成带有相对链接的输出。PDF 转换器或查看器会将所有 file:// 链接转换为绝对链接，这显然是不可行的。我希望未来能够找到解决此问题的方法。

指数

新的索引文件允许您通过单击任何标题对列进行排序。

出口结构

- 所有附件都位于同一个目录中
如果您使用像“附件”这样的固定名称，所有附件都将存放在该目录中。过去，这会生成“附件”、“附件-1”、“附件-2”等等目录。

- 附件与邮件保存在同一目录下。
在“文件名”选项卡上的这个新选项中，不会创建任何附件目录，因为它们都位于邮件目录中。

- 附件目录现在已设置邮件日期。
为了进行排序，附件目录的修改日期现在将设置为相应邮件的发送日期。

- 实验性的“瓦特”结构
这种结构会将每封邮件及其所有附件合并到格式化的附件目录中。典型的设置方法是将文件名格式设置为您偏好的格式，例如 ${subject}，并将目录格式也设置为 ${subject}。这样，每封邮件都会生成一个包含邮件正文和附件的目录。您可以按日期或名称对目录进行排序。

要尝试这种实验性结构，请输入以下单词：
withatts
在“其他”选项卡的日志记录选项中。（@Rayerd 建议）

日志记录选项

新的日志选项位于“其他”选项卡中。这些选项以空格分隔的日志类型单词列表形式设置。它们会生成各种日志和调试输出到调试控制台（可通过 Control-Shift-J 访问）。日志类型可以组合使用以生成累加输出。日志类型如下：
- msgs ：基本消息保存状态
- msgs2 ：更详细的消息调试输出
- summary ：出口概要
- msgparts ：MIME 消息部分的调试信息
- withatts ：一个临时选项，用于使用合并的消息和附件结构

ImportExportTools NG 概述和菜单

ImportExportTools NG 是基于 Paolo "Kaosmos" 的原创作品开发的后续扩展程序。该扩展程序提供多种导入和导出功能，支持多种不同的文件格式，包括：注意：并非所有功能都支持某些格式。
- mbox
- EML
邮件目录
- HTML
纯文本
- CSV（逗号分隔值）
PDF

虽然大部分功能都围绕邮件展开，但也支持导入和导出 Thunderbird 配置文件。此外，导出内容还可以包含 HTML 或 CSV 格式的索引文件。备份 Thunderbird 配置文件是另一项功能。

访问 IETng 功能是通过扩展程序安装的几个不同的菜单实现的。这些功能与上下文相关。新增的菜单包括：
- 文件夹窗格上下文（文件夹操作）
- 消息列表上下文（已选消息）
- 工具菜单（配置文件和备份操作）

“工具菜单”下的某些选项目前不在“汉堡菜单”中，这些选项将在未来的版本中添加。这些功能也可以在新工具栏按钮中找到。每个菜单和按钮都包含一个“帮助和选项”菜单项，用于访问选项对话框。

自定义日期格式

自定义日期选项允许您完全控制索引中两个文件的日期格式。文件名下拉菜单中同时提供默认格式“Date”和新的自定义格式“Date (custom)”。自定义日期还可以与扩展自定义文件名中使用的 ${date_custom} 标记一起使用。原始（默认）日期格式为：
YYYYMMDDHHmm : 202011060808

自定义日期格式使用 % 序列，其功能类似于 Unix 系统中的strftime()函数。它基于strftime()函数的 JavaScript 版本，该函数用于格式化日期和时间。
strftime() 的移植版本，作者：TH Doan (https://thdoan.github.io/strftime/)

新增和实验性偏好设置（选项）

有几个新功能和选项目前还无法在选项对话框中进行配置。这些功能稳定后，将会添加到用户界面中。
要使用这些功能，必须在配置编辑器中进行设置。要访问配置编辑器，请使用“工具/选项”菜单，然后选择“高级”部分，底部会有一个“配置编辑器”按钮。如果您在搜索栏中输入“importexporttoolsng”，即可找到所有扩展程序的首选项。然后，您可以根据需要编辑以下各项首选项。

importexporttoolsng.export.index_date_custom_format

此首选项遵循与索引文件相同的完整格式。默认情况下，它为空字符串；如果设置为格式化的日期/时间字符串，则会更改索引日期格式。

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

这两个设置选项都使用与导出邮件文件名相同的扩展文件命名格式。第一个选项用于外部附件，第二个选项用于嵌入式（内联）附件。

importexporttoolsng.experimental.use_delivery_date

通常情况下，Thunderbird 邮件的“日期”是发件人邮件客户端设置的日期。将此选项设置为“true”后，将使用收件人邮件客户端的接收日期。此日期将用于文件名以及索引文件中的日期。

基本全局热键（快捷键）功能

importexporttoolsng.experimental.hot_keys

这项新设置允许用户设置最多 10 个键盘快捷键。这些快捷键可以执行 ImportExportTools NG 的大部分功能以及其他 Thunderbird 功能。设置字符串采用 JSON 格式对象。格式和按键如下：

示例快捷键条目（JSON 对象数组）

id - 键索引（1-10）
key - 密钥（单个字符）
keycode - 'VK_*' 定义的键码
密钥或密钥代码只能选择一项，不能同时包含两者。
modifiers - 按键修饰键，以空格分隔的列表（'control'、'shift'、'alt'、'accel'）
oncommand - 直接命令字符串
contexts - 按键处于活动状态的窗口上下文
'all' ——指 Messenger、撰写、搜索下的所有窗口。
'messenger' ——主消息窗口及其衍生窗口
'compose' ——邮件撰写窗口
'search' ——搜索对话框
