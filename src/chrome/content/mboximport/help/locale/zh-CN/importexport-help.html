ImportExportTools NG 概述和菜单

ImportExportTools NG 是基于 Paolo“Kaosmos”原创作品的后续扩展。该扩展提供多种导入和导出功能，支持多种不同的文件格式，包括：注意：并非所有功能都支持某些格式。
- 盒子
- 电磁辐射
-邮件目录
- HTML
- 明文
- CSV（逗号分隔值）
- PDF

虽然大多数功能都围绕消息展开，但也支持导入和导出 Thunderbird 配置文件。此外，导出还可以包含 HTML 或 CSV 索引文件。Thunderbird 配置文件的备份是另一项功能。

通过扩展安装的几个不同菜单可以访问 IETng 功能。这些功能与上下文相关。添加了以下菜单：
- 文件夹窗格上下文（文件夹操作）
- 消息列表上下文（选定的消息）
- 工具菜单（配置文件和备份操作）

“工具菜单”下可访问的选项不在“汉堡”菜单中，它们将出现在未来的版本中。这些功能也可在新的工具栏按钮中使用。每个菜单和按钮还具有“帮助”和“选项”菜单项，用于访问选项对话框。

自定义日期格式

自定义日期选项允许完全控制索引中两个文件使用的日期格式。文件名下拉列表中提供默认格式“日期”以及新的自定义格式“日期（自定义）”。自定义日期还可以与扩展自定义文件名中使用的 ${date_custom} 标记一起使用。原始（默认）日期格式为：
YYYYMMDDHHmm : 202011060808

自定义日期格式利用与strftime() Unix 类功能相匹配的 % 序列。它基于strftime()的 JavaScript 端口，该函数用于格式化日期和时间。
TH Doan 移植的 strftime()（https://thdoan.github.io/strftime/）

转换序列说明

顺序 描述
%a 星期几的缩写名称。
%A 星期几的全名。
%b 月份名称的缩写。
%B 完整月份名称。
%c 当前区域设置的首选日期和时间 (UTC) 表示。
%C 世纪数（年/100）作为 2 位整数。
%d 以十进制数表示的月份日期（范围是 01 至 31）。
%e 以十进制数表示的月份日期（范围为 1 至 31）。
%F ISO 8601 日期格式（相当于%Y-%m-%d ）。
%G ISO 8601 基于周的年份，以十进制数表示世纪。4 位数字年份对应于 ISO 周数（参见%V ）。其格式和值与%Y相同，但如果 ISO 周数属于上一年或下一年，则使用该年份。
%g 与%G类似，但没有世纪，即使用 2 位数字的年份 (00-99)。
%H 使用 24 小时制的十进制数表示小时数（范围为 00 至 23）。另请参阅%k 。
%I 使用 12 小时制时钟（范围为 01 到 12）以十进制数表示的小时数。另请参阅%l 。
%j 一年中的第几天，以十进制数表示（范围是 001 至 366）。
%k 使用 24 小时制的十进制数表示小时数（范围为 0 到 23）。另请参阅%H 。
%l 使用 12 小时制（范围为 1 到 12）以十进制数表示的小时数。另请参阅%I 。
%m 月份为十进制数（范围为 01 至 12）。
%n 月份为十进制数（范围为 1 至 12）。
%M 分钟为十进制数（范围 00 至 59）。
%p 根据给定的时间值，可以是“AM”或“PM”。中午被视为“PM”，午夜被视为“AM”。
%P 类似于%p但采用小写形式（“am”或“pm”）。
%s 自纪元 1970-01-01 00:00:00 +0000 (UTC) 以来的秒数。
%S 以十进制数表示的秒数（范围是 00 至 59）。
%u 星期几以十进制表示（范围是 1 到 7），星期一为 1。另请参阅%w 。
%V ISO 8601 当前年份的周数，以十进制数表示（范围为 01 至 53），其中第 1 周是新年中至少有 4 天的第一个周（即第一个星期四）。
%w 星期几以十进制表示（范围是 0 到 6），星期日为 0。另请参阅%u 。
%x 当前语言环境的首选日期表示形式，不包括时间。
%X 当前语言环境的首选时间表示形式，不包含日期。
%y 年份为不带世纪的十进制数（范围为 00 至 99）。
%Y 年份为包含世纪的十进制数。
%z +hhmm或-hhmm数字时区（即与 UTC 的小时和分钟偏移量）。
%Z 时区名称或缩写。

新的和实验性的偏好设置（选项）

选项对话框中尚有几项新功能和选项无法配置。这些功能稳定后，它们将被添加到 UI 中。
为了使用它们，必须在配置编辑器中设置它们。要访问编辑器，请使用“工具/选项”菜单，然后选择“高级”部分，底部会有一个“配置编辑器”按钮。如果您在搜索栏中输入“importexporttoolsng”，您将获得所有扩展首选项。然后，您可以根据需要编辑下面的各种首选项。

importexporttoolsng.export.index_date_custom_format

此首选项遵循与索引文件相同的完整格式。默认情况下，它是一个空字符串，如果带有格式化的日期/时间字符串，则这将更改索引日期格式。

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

这两个首选项使用与导出邮件文件名相同的扩展文件命名格式。第一个用于外部附件，第二个用于嵌入式（内联）附件。

importexporttoolsng.experimental.use_delivery_date

通常，Thunderbird 中消息的“日期”是发件人的邮件客户端设置的日期。将此选项设置为“true”将使用收件人邮件客户端收到的日期。此日期将用于文件名以及索引文件中的日期。

基本、全局热键（快捷键）功能

importexporttoolsng.experimental.hot_keys

此新首选项允许用户设置最多 10 个键盘快捷键。这些快捷键可以执行大多数 ImportExportTools NG 功能以及其他 Thunderbird 功能。首选项字符串的格式为 JSON 格式的对象。格式和键如下：

modifiers - 键修饰符，以空格分隔的列表（'control'、'shift'、'alt'、'accel'）

'messenger' ——主 messenger 窗口及其子窗口
