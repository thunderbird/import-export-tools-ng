Note di rilascio: Nuova importante versione v15

Note, funzionalità e modifiche della versione 15. Si prega di leggere attentamente.

La versione 15 rappresenta una completa riscrittura delle funzioni di esportazione dei messaggi, sia per le cartelle che per i messaggi selezionati. È la prima fase di un'implementazione basata principalmente su una tecnologia WebExtension moderna. Il nuovo design offre un notevole miglioramento architetturale, consentendo l'introduzione di nuove funzionalità e prestazioni superiori.

Per la prima versione 15, sono stati implementati i seguenti formati:

- EML
- HTML e allegati
- PDF e allegati
- Testo semplice e allegati

Oltre alla nuova funzionalità di esportazione in PDF con allegati, sono disponibili anche una nuova finestra di stato dell'esportazione e una nuova alternativa alle notifiche del sistema operativo. Di seguito un elenco delle nuove funzionalità:

Esportazione in PDF con allegati

Questa è un'opzione richiesta da tempo. C'è un'importante limitazione rispetto all'esportazione in HTML: la tabella degli allegati non contiene link cliccabili. Finora non è stato possibile produrre un output con link relativi. Sia il convertitore PDF che i visualizzatori convertono tutti i link file:// in link assoluti, che risultano inutilizzabili. Spero che in futuro si trovi una soluzione a questo problema.

Indice

Il nuovo file indice consente di ordinare le colonne facendo clic su una qualsiasi delle intestazioni.

Strutture di esportazione

- Tutti gli allegati in un'unica cartella
Se si utilizza una parola statica come "Allegati", tutti gli allegati verranno inseriti in quella cartella. In passato, ciò avrebbe generato cartelle come Allegati, Allegati-1, Allegati-2 e così via.

- Gli allegati vengono salvati nella stessa cartella dei messaggi.
In questa nuova opzione, disponibile nella scheda Nomi file, non vengono create cartelle per gli allegati, poiché questi sono tutti contenuti nella cartella dei messaggi.

- Le cartelle degli allegati ora hanno la data del messaggio impostata
Ai fini dell'ordinamento, le cartelle degli allegati avranno ora la data di modifica impostata sulla data di invio del messaggio corrispondente.

- Struttura sperimentale "withatts"
Questa struttura combina ogni messaggio con tutti gli allegati in una directory di allegati formattata. La configurazione tipica prevede di impostare il formato del nome file secondo le proprie preferenze, ad esempio ${subject}, e di impostare anche il formato della directory su ${subject}. In questo modo, ogni messaggio verrà creato in una directory contenente sia il messaggio stesso che gli allegati. È possibile ordinare le directory per data o per nome.

Per provare questa struttura sperimentale, inserisci la parola:
withatts
nelle opzioni di registrazione nella scheda Varie. (Proposta di @Rayerd)

Opzioni di registrazione

Le nuove opzioni di registrazione si trovano nella scheda Varie. Sono impostate come un elenco di parole che indicano i tipi di log, separate da spazi. Producono vari output di log e di debug nella console di debug, accessibile tramite la combinazione di tasti Ctrl+Maiusc+J. I tipi di log possono essere combinati per produrre un output additivo. I tipi di log sono i seguenti:
- msgs : Stato di salvataggio del messaggio di base
- msgs2 : output di debug più dettagliato per i messaggi
- summary : Un riepilogo dell'esportazione
- msgparts : Informazioni di debug per le parti del messaggio MIME
- withatts : un'opzione temporanea per utilizzare la struttura combinata di messaggi e allegati

Menù

Le nuove funzioni vengono attualmente aggiunte in aggiunta alle funzioni preesistenti e hanno il prefisso (v15).

Il vecchio codice verrà deprecato una volta che le nuove funzioni saranno stabili. Questa scelta è stata fatta per precauzione in vista del rilascio della versione principale. Si prega di utilizzare le nuove funzioni della versione 15 e di segnalare eventuali problemi nel repository GitHub qui:

Panoramica e menu di ImportExportTools NG

ImportExportTools NG è un'estensione derivata dal lavoro originale di Paolo "Kaosmos". L'estensione offre molteplici funzioni di importazione ed esportazione che supportano diversi formati di file, tra cui: Nota: alcuni formati non sono supportati da tutte le funzioni.
- mbox
- EML
- maildir
- HTML
- testo semplice
- CSV (Valori separati da virgola)
- PDF

Sebbene la maggior parte delle funzionalità ruoti attorno ai messaggi, è presente anche il supporto per l'importazione e l'esportazione dei profili Thunderbird. Inoltre, le esportazioni possono includere anche un file indice HTML o CSV. Un'altra funzionalità disponibile è il backup del profilo Thunderbird.

L'accesso alle funzionalità di IETng avviene tramite diversi menu installati dall'estensione. Le funzioni sono sensibili al contesto. I menu aggiunti sono:
- Contesto del riquadro delle cartelle (operazione sulle cartelle)
- Contesto dell'elenco dei messaggi (messaggi selezionati)
- Il menu Strumenti (operazioni di profilo e backup)

Le opzioni accessibili tramite il "Menu Strumenti" non sono presenti nel menu "Hamburger", ma lo saranno in una versione futura. Queste funzioni sono disponibili anche tramite il nuovo pulsante della barra degli strumenti. Ciascun menu e pulsante dispone inoltre di una voce di menu "Aiuto e Opzioni" per accedere alla finestra di dialogo delle opzioni.

Supporto:

Segnala eventuali problemi o richieste di nuove funzionalità nel repository GitHub dell'estensione: ImportExportTools NG Issues

In caso di segnalazioni di bug, si prega di includere le seguenti informazioni per facilitare la risoluzione del problema:
- Versione ImportExportTools NG
- Versione Thunderbird
- Sistema operativo e versione
- Output di debug (copia dalla console di debug - accesso tramite Control-Shift-J)
- Descrizione delle fasi che hanno portato al problema

Formato data personalizzato

L'opzione data personalizzata consente il controllo completo sulla formattazione della data utilizzata per entrambi i file nell'indice. Sia il formato predefinito "Data" che il nuovo formato personalizzato "Data (personalizzata)" sono disponibili nei menu a tendina dei nomi dei file. La data personalizzata può essere utilizzata anche con il token ${date_custom} utilizzato nei nomi dei file personalizzati estesi. Il formato data originale (predefinito) è:
AAAAMMGGHmm : 202011060808

Il formato data personalizzato utilizza sequenze % che corrispondono alla funzionalità di tipo Unix della funzione strftime() . Si basa su una versione JavaScript di strftime() , una funzione per formattare data e ora.
Versione della funzione strftime() realizzata da TH Doan (https://thdoan.github.io/strftime/)

%A Nome completo del giorno della settimana.
%b Nome abbreviato del mese.
%B Nome completo del mese.

%g Simile a %G , ma senza secolo, ovvero con un anno a due cifre (00-99).
%H Ora espressa come numero decimale utilizzando il formato orario a 24 ore (intervallo da 00 a 23). Vedi anche %k .

%l Ora espressa come numero decimale utilizzando il formato orario a 12 ore (intervallo da 1 a 12). Vedi anche %I .

  • %c - la stringa formattata è leggermente diversa
  • %D - non implementato (utilizzare %m/%d/%y o %d/%m/%y )
  • %e - lo spazio non viene aggiunto
  • %E - non implementato
  • %h - non implementato (utilizzare %b )
  • %k - lo spazio non viene aggiunto
  • %n - come %m , ma senza zero iniziale (usa \n per la nuova riga)
  • %O - non implementato
  • %r - non implementato (utilizzare %I:%M:%S %p )
    • ${subject} L'oggetto del messaggio
  • %R - non implementato (utilizzare %H:%M )
  • %t - non implementato (utilizzare \t )
  • %T - non implementato (utilizzare %H:%M:%S )
  • %U - non implementato
  • %W - non implementato
  • %+ - non implementato
  • %% - non implementato (utilizzare % )

Preferenze nuove e sperimentali (opzioni)

Sono presenti diverse nuove funzionalità e opzioni non ancora configurabili tramite la finestra di dialogo delle opzioni. Una volta stabilizzate, queste funzionalità verranno aggiunte all'interfaccia utente.
Per poterle utilizzare, è necessario configurarle nell'Editor di configurazione. Per accedere all'editor, vai al menu Strumenti/Opzioni, seleziona la sezione Avanzate e in basso troverai il pulsante "Editor di configurazione". Se digiti "importexporttoolsng" nella barra di ricerca, visualizzerai tutte le preferenze dell'estensione. Potrai quindi modificare le varie preferenze sottostanti a seconda delle tue esigenze.

importexporttoolsng.export.index_date_custom_format

Questa preferenza segue la stessa formattazione completa utilizzata per i file di indice. Di default è una stringa vuota; se contiene una stringa di data/ora formattata, modificherà il formato della data nell'indice.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Entrambe queste preferenze utilizzano lo stesso tipo di formato di denominazione estesa dei file utilizzato per i messaggi esportati. La prima è per gli allegati esterni, la seconda per gli allegati incorporati (in linea).

importexporttoolsng.experimental.use_delivery_date

Normalmente, la "Data" di un messaggio in Thunderbird corrisponde alla data impostata dal client di posta del mittente. Impostando questa opzione su "true", verrà utilizzata la data di ricezione del messaggio nel client di posta del destinatario. Questa data verrà utilizzata sia per i nomi dei file che per la data nei file di indice.

Funzione di base dei tasti di scelta rapida (scorciatoie) globali

importexporttoolsng.experimental.hot_keys

Questa nuova preferenza consente all'utente di impostare fino a 10 scorciatoie da tastiera. Le scorciatoie possono eseguire la maggior parte delle funzioni di ImportExportTools NG, nonché altre funzioni di Thunderbird. La stringa della preferenza è formattata come un oggetto in formato JSON. Il formato e le chiavi sono i seguenti:

Esempi di tasti di scelta rapida (array di oggetti JSON)

id - l'indice della chiave (1-10)
key - la chiave (singolo carattere)
keycode - il codice tasto definito 'VK_*'
È possibile includere la chiave o il codice di attivazione, ma non entrambi.
modifiers - i tasti modificatori, elenco separato da spazi ('control', 'shift', 'alt', 'accel')
oncommand - una stringa di comando diretta
contexts - il contesto della finestra in cui il tasto è attivo
'all' - tutte le finestre sotto messenger, componi, cerca
'messenger' - la finestra principale di messaggistica e i suoi discendenti
'compose' - la finestra di composizione del messaggio
